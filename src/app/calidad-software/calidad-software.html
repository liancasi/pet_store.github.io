Fundamentos de Calidad de Software

Principios, normas y modelos de calidad aplicados al proyecto PetStore

📖

Descripción General

Establece las bases conceptuales de la calidad de software aplicada al proyecto PetStore. Aborda estándares internacionales, modelos de calidad y métricas iniciales con base en ISO/IEC 25010.

ISO/IEC 25010 Modelos de Calidad Atributos de Calidad Métricas Iniciales

Calidad Funcional

  • •Completitud funcional de los módulos
  • •Exactitud en el registro de datos clínicos
  • •Adecuación funcional para veterinarios
  • •Interoperabilidad entre módulos

Calidad No Funcional

  • •Fiabilidad del sistema en producción
  • •Usabilidad para personal veterinario
  • •Eficiencia en tiempos de respuesta
  • •Mantenibilidad y portabilidad del código
Meta de cobertura QA: El proyecto establece cobertura de pruebas ≥ 80% y cero errores críticos en producción al lanzamiento de la versión Beta.

Código Limpio: Principios, Prácticas y Métricas de Calidad

SOLID · DRY · Complejidad Ciclomática · SonarQube · Refactoring

🧱

Principios SOLID y DRY en PetStore

PrincipioDescripciónAplicación en PetStore
S — SRPSingle Responsibility PrincipleCada servicio tiene una única responsabilidad bien definida
O — OCPOpen/Closed PrincipleLa arquitectura permite agregar módulos sin modificar existentes
L — LSPLiskov Substitution PrincipleImplementaciones de interfaces son intercambiables
I — ISPInterface Segregation PrincipleInterfaces separadas por dominio (mascotas, usuarios, facturación)
D — DIPDependency Inversion PrincipleInyección de dependencias en Spring Boot para desacoplar capas
DRYDon't Repeat YourselfEliminación de código duplicado en validaciones y transformaciones

Clases Limpias

  • •Alta cohesión: atributos y métodos relacionados

Concurrencia y Refinamiento

  • •Transacciones (@Transactional) para consistencia
  • •Gestión segura de acceso concurrente

  • •Code reviews periódicas del equipo

Pruebas de Software

TDD · JUnit 5 · JaCoCo 0.8.11 · Pruebas Unitarias, Integración y Automatizadas

🧪

Tipos de Pruebas de Software

Pruebas Unitarias

  • •Verifican unidades individuales (métodos, clases)
  • •Aisladas con mocking (Mockito)
  • •Implementadas con JUnit 5 en Spring Boot
  • •Ejecutadas automáticamente en pipeline CI

Caja Blanca · Gris · Negra

  • •Blanca: lógica interna, ramas y condiciones Java
  • •Gris: endpoints REST + respuesta de base de datos
  • •Negra: flujos de usuario en la interfaz web (UAT)
🔁

TDD – Test-Driven Development

🔴
Red

Escribir la prueba que falla primero — define el comportamiento esperado

🟢
Green

Implementar el código mínimo para que la prueba pase

♻️
Refactor

Mejorar el código manteniendo todas las pruebas en verde

⚙️
